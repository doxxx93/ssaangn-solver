# 쌍근 솔버 (ssaangn-solver)

[쌍근](https://ssaangn.com/) — Portal 개발자 Jeep Barnett가 만든 한글 2글자 워들 — 을
푸는 Rust 솔버. 추측과 그때 받은 이모지 힌트를 입력하면 남은 후보와
다음에 칠 추측을 추천한다.

**🌐 웹에서 바로 쓰기: <https://doxxx93.github.io/ssaangn-solver/>**

같은 솔버 코어를 WebAssembly로 빌드해 브라우저에서 돌린다 (`src/lib.rs` + `web/`).
서버 없이 페이지에서 전부 계산하므로 폰에서도 그대로 쓸 수 있다. 아래는 터미널 TUI 버전.

## 규칙 모델

각 글자를 자모로 분해해(초성/중성/종성), 같은 위치끼리 비교한다. 복합모음·겹받침은
쪼개고(ㅘ→ㅗㅏ, ㅄ→ㅂㅅ …), 초성 쌍자음은 안 쪼갠다. 분해 규칙은 게임 원본
`hangul_tools.js`를 그대로 포팅했다 (`src/hangul.rs`).

힌트 6종 (`src/matcher.rs`):

| 이모지 | 의미 | 판정 |
|---|---|---|
| 🥕 | 당연하죠 | 그 글자와 완전 일치 |
| 🍄 | 비슷해요 | 자모 2개 이상 일치 **+ 초성도 일치** |
| 🧄 | 많을 거예요 | 자모 2개 이상 일치, 초성은 불일치 |
| 🍆 | 가지고 있어요 | 자모 1개만 일치 |
| 🍌 | 반대로요 | 같은 위치엔 없고 **반대쪽 글자**에 있음 |
| 🍎 | 사과 | 두 글자 어디에도 없음 |

순방향 매처는 라이브 서버(`get_external_clues.php`)와 랜덤 160쌍 대조에서 **100% 일치**
검증됨 (`scripts/validate_matcher.py`).

## 사용법

```bash
cargo run --release            # TUI 실행
```

입력 한 줄에 `<추측> <피드백2개>` 형식. 피드백은 게임 아이콘 이름으로 입력한다:

| 입력 | 이모지 | 뜻 |
|---|---|---|
| `당근` | 🥕 | 그 글자와 완전 일치 |
| `버섯` | 🍄 | 자모 2개+ 일치, 초성도 일치 |
| `마늘` | 🧄 | 자모 2개+ 일치, 초성은 불일치 |
| `가지` | 🍆 | 자모 1개 일치 |
| `바나나` | 🍌 | 반대쪽 글자에 있음 |
| `사과` | 🍎 | 어디에도 없음 |

```
관심 가지 사과       # 띄어쓰기 있어도/없어도 됨: 관심 가지사과
```

이모지(`관심 🍆🍎`)나 숫자(`관심 46`, 1=당근 … 6=사과)도 그대로 받는다.

**확정된 글자/자모를 위치로 직접 입력**할 수 있다 (추론으로 알아낸 것도 반영):

| 입력 | 뜻 |
|---|---|
| `1복` | 첫 글자가 확정 = 복 |
| `2용` | 둘째 글자가 확정 = 용 |
| `1ㅂ` | 첫 글자에 자모 ㅂ 포함 |
| `2ㅇ` | 둘째 글자에 자모 ㅇ 포함 |

🎃 호박 힌트(자음·모음 1개 공개)도 입력할 수 있다. `h ㅏ` (또는 `🎃ㅏ`) →
그 자모가 정답 두 글자 중 어딘가에 있다는 제약으로 후보를 좁힌다.

**힌트는 첫 추측 직후에 쓰는 게 가장 좋다.** 공짜(추측 미소모)라 빠를수록 이득이
모든 후속 추측에 누적된다. 880게임 시뮬(opening 관심, `ssaangn hintsim`):

| 타이밍 | 평균 시도 |
|---|---:|
| 안 씀 | 4.13 |
| 첫 추측 전 | 3.71 |
| **1수 후** | **3.69** |
| 2수 후 | 3.73 |
| 3수 후 | 3.90 |
| 4수+ | 효과 없음 |

첫 추측 전엔 공개 자모가 오프닝과 겹쳐 약간 손해, 3수 이후엔 이미 후보가 좁아져
늦는다. 1~2수 사이가 스윗스폿.

- `u` 직전 입력 취소 · `r` 초기화 · `q`/Esc 종료
- `x <단어>` 정답이 될 수 없는 junk 후보(예: 음역 고유명사)를 영구 제외한다. 양쪽 풀에서
  빼고 `data/blocklist.json`에 누적해 다음 실행부터 자동 적용. 폴백에서 가끔 보이는
  잔여 비단어를 쓸수록 깨끗하게 만든다.
- 추천 추측: **"기대 잔여 후보"(1수 정보량)가 작은 순**. 정보탐욕 1-ply 그리디다 —
  게임의 7수 제한 때문에, 평균을 줄이려 7수 절벽 너머로 도박하는 룩어헤드보다 후보를
  고르게 쪼개는 1-ply가 패배를 적게 낸다(out-of-pool 시뮬 패배 11/300→2/300). 동률이면
  *이번 턴에 맞출 수 있는* 정답 후보(`*`)를, 그다음 흔한 단어를 위로. `*` = 그 단어 자체가
  아직 정답일 수 있다는 뜻.
- 점수는 **풀별로 다르게**: 1차풀은 정답이 거의 균등하니 균등 채점, 폴백은 흔한단어~junk가
  섞여 prior 정보량이 크니 **빈도가중 채점**(흔한 정답을 먼저 해소). 채점은 `rayon`으로 병렬.
- 남은 후보 목록은 **빈도순**(흔한 답이 먼저). 정답일 가능성이 높은 순서다.
- `ssaangn analyze <정답> <추측...>` / `ssaangn replay`(stdin에 TUI 입력)로 분석할 수 있다.
  `ssaangn sim`은 정답이 1차풀 밖이면 폴백으로 계속 풀어 out-of-pool 성능까지 측정한다.

### 권장 오프닝

과거 정답 880개를 그리디로 끝까지 푸는 시뮬레이션(`ssaangn sim <단어>`) 평균 시도수:

| 오프닝 | 평균 시도 |
|---|---|
| **관심** | **4.14** |
| 확신 | 4.15 |
| 학원 | 4.16 |
| 남성 | 4.22 |
| 신발 | 4.22 |
| 간장 | 4.28 |

**관심**(ㄱㅗㅏㄴ/ㅅㅣㅁ, 고유 자모 7개)이 가장 빠르다. 1-ply "기대 잔여" 지표만 보면
간장이 1위지만, ㅏ가 중복돼 실전 풀이에선 살짝 손해다. `ssaangn openings 20`으로
1-ply 랭킹을 볼 수 있다.

## 데이터

- `data/candidates.json` (2,376) — 1차 정답 후보 풀. ssaangn 과거 정답 + 한들(handle)
  정답 사전의 합집합. 흔한 답 위주라 추천 품질이 좋다.
- `data/words.json` (97,330) — 유효 추측 검증용 (한들 단어 사전 ∪ 후보).
- `data/fallback.json` (12,362) — 1차 풀이 비면(정답이 풀 밖) 쓰는 폴백. `data/words.json`
  중 **2글자 명사만** 남긴 것 ∪ 과거 정답, 다시 **자막빈도>0만** 남겨 정리했다(53,744→12,362).
  빈도0 명사는 `옌병·원평`처럼 게임이 추측으론 받아도 정답은 안 되는 음역 고유명사/비단어라
  distractor만 늘렸기 때문. (정답은 `PROVEN_FLOOR`로 floor돼 생존.)
- `data/weights.json` — 단어→빈도 가중치. 남은 후보를 **빈도순(흔한 답 먼저)**으로 보여주고,
  폴백 채점·추천 동률에 쓴다.
- `data/blocklist.json` — `x <단어>`로 누적하는 사용자 제외 목록(런타임 파일). 시작 시 적용.
- `data/recent_answers.json` — DCInside 2026년 5월 일별 정답.
- 재생성: `python3 scripts/build_data.py` (1차 풀) → `python3 scripts/build_freq.py`
  (폴백·가중치). 후자는 OpenSubtitles 한국어 빈도(`hermitdave/FrequencyWords`의
  `content/2018/ko/ko_full.txt`)를 `/tmp/ko_full.txt`에 받아두고, `kiwipiepy`로 명사
  태깅한다(`pip install kiwipiepy`). 명사 태깅 결과는 `data/raw/valid_nouns.json`에 캐시.

> 동작: 정답이 1차 풀(2,376) 안이면 깔끔하게 좁힌다(과거 880개 시뮬 평균 4.13회, 전부
> 7회 이내). 풀 밖이면 명사 폴백으로 자동 전환해 빈도순 후보를 보여준다.

## 검증

```bash
cargo test                     # 분해/매처 단위 테스트 (게임 예시 포함)
./target/release/ssaangn sim < secrets.txt   # 정답 리스트로 그리디 시뮬레이션
python3 scripts/validate_matcher.py 160      # 라이브 서버 대조 (정중히, rate-limited)
```
